# AirVibe Integrators Wiki
An internal documentation platform for the Machine Saver team, built around the AirVibe LoRaWAN Condition Monitoring Sensor.

## ЁЯЪА Overview
The AirVibe Integrators Wiki is a dedicated knowledge hub where technicians, integrators, and business development teammates can access up-to-date reference material for AirVibe sensors.
It's designed for:
- Collaborative updating of integration procedures and field tips
- Rapid onboarding of new partners or internal staff
- Version-tracked documentation that reflects our evolving product suite

## ЁЯз▒ Key Features
- **Sidebar navigation** тАФ flat section-based nav (Overview, Quickstart, Uplinks, Downlinks, Config & Modes, Alarms, Time Waveform, Firmware Updates, Codec Utility)
- **AI-powered search** тАФ ask questions in plain English; answers are generated by Google Gemini with context drawn from the full wiki content
- **Interactive tools** тАФ built-in uplink decoder, downlink encoder, alarm bitmask calculator, waveform tracker, and FUOTA helper

## ЁЯПЧя╕П Architecture & Deployment
- **Stack:** React + TypeScript, built with Vite. Tailwind CSS via CDN. Mermaid.js diagrams via CDN import map.
- **Hosting:** GitHub Pages at https://airvibe-wiki.machinesaver.com/
- **CI/CD:** Automated deploy via GitHub Actions on push to `main`. Requires a `GEMINI_API_KEY` repository secret.

```bash
npm install       # Install dependencies
npm run dev       # Dev server at http://localhost:3000
npm run build     # Production build to dist/
npm run preview   # Preview production build locally
```

## ЁЯдЭ Contributing Guidelines
1. Fork the repo and create a feature branch (`feature/your-topic`).
2. Content lives in TypeScript files under `data/sections/` тАФ one file per section (e.g. `uplinks.ts`, `alarms.ts`). Edit the relevant file to add or update a `WikiPage` entry.
3. Mermaid diagrams go in the `mermaidDiagram` field of a `WikiPage`. No separate asset files are needed.
4. Run `npm run build` locally to confirm the build passes before submitting.
5. Submit a Pull Request.

## ЁЯза Tips for Effective Use
- Use the AI search bar to ask questions in natural language (e.g., "What is the TWF packet structure?" or "How do I configure alarm thresholds?").
- When writing new pages: aim for "What you need to know + How to do it + Common issues".
- Keep revision summaries short but meaningful (e.g., "Updated LoRaWAN downlink table for firmware v2.1").
